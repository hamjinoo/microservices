# 스프링부트 마이크로서비스 학습 로드맵

> **프론트엔드 개발자에서 풀스택 개발자로!**  
> 실무에서 바로 사용할 수 있는 스프링부트 마이크로서비스 아키텍처를 단계별로 학습합니다.

---

## 📚 이 로드맵의 목표

1. **백엔드의 기본 동작 원리 이해**: 프론트엔드에서 호출하는 API가 어떻게 만들어지는지 이해
2. **실무 수준의 API 개발**: CRUD를 넘어 인증, 예외 처리, 트랜잭션까지 
3. **마이크로서비스 아키텍처**: 모놀리식에서 MSA로의 전환 경험
4. **배포까지 경험**: Docker, Kubernetes를 활용한 실제 배포

---

## 🎯 학습 대상

- **프론트엔드 개발 경험이 있는 분**
- **백엔드 개념은 알지만 실제 구현 경험이 부족한 분**
- **API를 호출만 해봤지, 만들어본 적은 없는 분**

## 🗺️ 10단계 학습 로드맵

### Phase 1: 기초 다지기 (1-2단계)
**목표**: 스프링부트의 기본 동작 원리를 이해하고 간단한 CRUD API를 만들 수 있다.

#### Step 1: 스프링부트 기초
- **학습 시간**: 1-2일
- **핵심 개념**: IoC/DI, 스프링 컨테이너, 의존성 주입
- **실습**: 개발 환경 설정, Hello World API 만들기
- **실생활 비유**: 새 집에 입주하기 전에 전기 차단기와 수도 밸브 위치를 확인하듯, 애플리케이션의 전원 스위치와 배선 구조를 이해합니다.

#### Step 2: 기본 게시판 API 개발
- **학습 시간**: 3-5일
- **핵심 개념**: Entity-Repository-Service-Controller 계층, JPA, REST API
- **실습**: 게시글 CRUD API 구현, H2 데이터베이스 연동
- **실생활 비유**: 회사가 상담창구, 해결팀, 기록보관실로 역할을 나누듯이 각 계층의 책임을 분리해 협업하기 쉬운 구조를 만듭니다.

**이 단계를 마치면**: Postman으로 CRUD API를 테스트할 수 있고, 데이터베이스에 저장된 내용을 확인할 수 있습니다.

---

### Phase 2: 실무 수준으로 업그레이드 (3-4단계)
**목표**: 예외 처리, 검증, 인증 등 실무에서 필수적인 기능을 추가한다.

#### Step 3: 게시판 기능 확장
- **학습 시간**: 3-4일
- **핵심 개념**: 연관관계 매핑, 페이징, 검색, DTO 패턴
- **실습**: 댓글 기능, 페이징, 검색 기능 추가
- **실생활 비유**: 도서관이 잡지를 10권씩 묶어 진열하고 인덱스 카드를 활용해 원하는 책만 꺼내오듯, 데이터를 보기 좋게 나누고 필요한 정보만 추려 제공합니다.

#### Step 4: 인증 및 권한 관리
- **학습 시간**: 5-7일
- **핵심 개념**: Spring Security, JWT, 역할 기반 접근 제어
- **실습**: 회원가입, 로그인, 권한별 API 접근 제어
- **실생활 비유**: 건물 출입증을 발급해 신원 확인과 권한을 동시에 확인하듯, 토큰을 활용해 누가 어느 구역까지 들어올 수 있는지 관리합니다.

**이 단계를 마치면**: 로그인이 필요한 API, 관리자만 접근 가능한 API 등을 만들 수 있습니다.

---

### Phase 3: 마이크로서비스로 전환 (5-7단계)
**목표**: 모놀리식 애플리케이션을 마이크로서비스로 분리하고 서비스 간 통신을 구현한다.

#### Step 5: 마이크로서비스 전환 - 1
- **학습 시간**: 5-7일
- **핵심 개념**: MSA 아키텍처, 서비스 분리, Service Discovery, Config Server
- **실습**: 게시판을 User Service, Board Service, Comment Service로 분리
- **실생활 비유**: 대형마트 안에 모든 코너가 있는 대신, 거리마다 빵집·약국·미용실이 따로 운영되는 상가처럼 기능을 전문 매장 단위로 나눕니다.

#### Step 6: 마이크로서비스 전환 - 2
- **학습 시간**: 5-7일
- **핵심 개념**: API Gateway, Feign Client, Circuit Breaker
- **실습**: API Gateway 구축, 서비스 간 통신, 장애 대응
- **실생활 비유**: 아파트 경비실이 방문객을 확인하고 해당 동으로 안내하듯, 게이트웨이가 모든 요청을 한 번에 검문하고 올바른 서비스로 연결합니다.

#### Step 7: 이벤트 기반 아키텍처
- **학습 시간**: 5-7일
- **핵심 개념**: 메시지 브로커(Kafka/RabbitMQ), 이벤트 프로듀서/컨슈머
- **실습**: 게시글 생성 시 알림 이벤트 발행, 이메일 서비스에서 이벤트 소비
- **실생활 비유**: 택배 배송 알림이나 문자 공지처럼, 어떤 사건이 발생하면 관련 부서에 자동으로 연락이 전달되도록 설계합니다.

**이 단계를 마치면**: 독립적으로 배포 가능한 여러 개의 서비스를 만들고 연결할 수 있습니다.

---

### Phase 4: 프로덕션 준비 (8-10단계)
**목표**: 테스트, 배포, 모니터링 등 실제 운영에 필요한 모든 것을 경험한다.

#### Step 8: 테스트 및 배포
- **학습 시간**: 5-7일
- **핵심 개념**: JUnit, Mockito, Docker, Kubernetes
- **실습**: 단위/통합 테스트 작성, Docker 이미지 생성, Kubernetes 배포
- **실생활 비유**: 자동차 공장에서 조립이 끝난 차량을 검사하고, 표준화된 공정으로 신차를 연속 생산하는 과정과 같습니다.

#### Step 9: 모니터링 및 최적화
- **학습 시간**: 5-7일
- **핵심 개념**: 로깅, 성능 최적화, 캐싱(Redis), N+1 문제 해결
- **실습**: ELK 스택으로 로그 수집, Redis 캐싱, 쿼리 최적화
- **실생활 비유**: 관제센터에서 CCTV와 계측기를 보며 문제를 조기에 발견하고, 인기 상품을 계산대 옆에 쌓아 두는 편의점처럼 성능 병목을 줄입니다.

#### Step 10: 최종 프로젝트 완성
- **학습 시간**: 7-10일
- **핵심 개념**: 전체 통합, 프론트엔드 연동, CI/CD
- **실습**: React/Vue로 프론트엔드 개발, 전체 시스템 통합, GitHub Actions로 자동 배포
- **실생활 비유**: 여러 공정이 연결된 자동화 공장에서 생산, 포장, 품질검사까지 한 번에 진행하듯 전체 시스템을 묶어 완성품을 내보냅니다.

**이 단계를 마치면**: 실무에서 바로 사용할 수 있는 완전한 마이크로서비스 시스템을 구축할 수 있습니다.

## 💡 학습 팁

### 실생활 관점에서 백엔드 이해하기

| 백엔드 개념    | 실생활 비유                                            |
|---------------|--------------------------------------------------------|
| Controller    | 민원 창구에서 시민을 맞이하는 상담 직원               |
| Service       | 상담 내용을 바탕으로 해결책을 찾는 내부 실무팀        |
| Repository    | 기록 보관실에서 필요한 서류를 찾아오는 기록 담당      |
| Entity        | 정보를 기록하는 서류 양식 한 장                        |
| DTO           | 고객에게 전달하는 요약 보고서와 체크리스트            |
| DI Container  | 필요한 물품을 제때 제공하는 중앙 창고와 관리자        |
| @Transactional| 문제가 생기면 계약서를 원래대로 되돌리는 안전장치     |
| JWT           | 출입증처럼 신원을 증명하고 권한을 확인하는 카드        |
| API Gateway   | 건물 정문에서 방문자를 확인하고 안내하는 경비실       |
| Message Queue | 사건이 발생하면 관련자에게 순차적으로 전달되는 알림망 |

## 🆘 막혔을 때

### 자주 발생하는 문제
- **의존성 오류**: `build.gradle` 또는 `pom.xml` 확인
- **포트 충돌**: 다른 애플리케이션이 8080 포트 사용 중
- **데이터베이스 연결 오류**: `application.properties` 설정 확인
- **빌드 실패**: `./gradlew clean build` 실행

### 도움 받기
1. **에러 메시지 구글링**: 영어로 검색하면 Stack Overflow 결과가 많음
2. **공식 문서**: Spring.io 공식 문서
3. **커뮤니티**: OKKY, Inflearn 질문 게시판
